## Full Content

- Check 'Proven Wins' before implementing strategy to see if validated
- Check 'Failed Tests' before repeating pattern to avoid known pitfalls
- Add successful tests to 'Proven Wins' and identify rollout candidates after validation
- Add failed tests to 'Failed Tests' with learnings after negative results
- Document original test client, date, change, and reasoning
- Define clear use cases for when pattern should be applied
- Track rollout status across clients for validated strategies
- Create implementation checklists for repeatable patterns
- Monitor for 14-28 days before declaring structural changes validated
- Identify rollout candidates across client portfolio for proven wins
- Split product lines into standalone campaigns when they represent >15% revenue with distinct seasonality
- Create category-based asset groups for clients with 5-8 primary categories
- Use historical performance to set baseline ROAS targets for new campaigns
