## Infrastructure: Fix ANTHROPIC_API_KEY Auth Conflict (Cost: $25)
**Completed:** 2025-11-20 15:30
**Source:** Manual completion (reported in Claude Code)

**Issue:** ANTHROPIC_API_KEY was set globally in `~/.bashrc`, causing Claude Code interactive sessions to use API billing instead of the claude.ai token. This resulted in unexpected charges of **$25 today** for what should have been free/included usage.

**Root Cause:**
- `export ANTHROPIC_API_KEY="sk-ant-api03-..."` was present in `~/.bashrc`
- Origin unknown (not intentionally added)
- Claude Code prioritised API key over claude.ai token for authentication
- All interactive Claude Code work today was billed to API account instead of claude.ai subscription
- Warning visible in Claude Code: "âš Auth conflict: Both a token (claude.ai) and an API key (ANTHROPIC_API_KEY) are set"

**Cost Impact:**
- **$25 charged today** for interactive sessions that should have used claude.ai token
- Background agents (daily-intel-report, ai-inbox-processor, etc.) are supposed to use API key explicitly
- Interactive work (ad-hoc analysis, client work, conversations) should use claude.ai token (included in subscription)

**Solution Implemented:**
1. Removed `export ANTHROPIC_API_KEY=...` line from `~/.bashrc`
2. Unset in current session (requires Claude Code restart to fully apply)
3. Background agents still work - they explicitly set API key in their LaunchAgent plist files

**Files Modified:**
- `/Users/administrator/.bashrc` - Removed ANTHROPIC_API_KEY export

**Prevention:**
- Monitor Anthropic API usage dashboard for unexpected spikes
- Verify auth method on Claude Code startup (should only see claude.ai, not API key)
- Document that API key should ONLY be set in agent-specific LaunchAgent plists, never in shell profile
- Keep API key confined to background automation, not interactive sessions

**Authentication Architecture (Correct Setup):**
- **Interactive Claude Code:** Uses claude.ai token (included in subscription)
- **Background Agents:** Use ANTHROPIC_API_KEY set in LaunchAgent plist `EnvironmentVariables`
- **Shell Profile:** Should NOT have ANTHROPIC_API_KEY set globally

**Lesson:** Global environment variables in shell profiles can cause expensive authentication routing issues. Keep API credentials scoped to specific processes.
